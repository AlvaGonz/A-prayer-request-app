# User Types & Top User Stories

## User Types

| Type | Description |
| :--- | :--- |
| **Guest** | An unregistered visitor who can browse the wall, post anonymous requests, and tap "I prayed". No persistent identity. |
| **Registered Member** | A signed-up user with email + password. Can post named or anonymous requests, receive notifications, and manage their own requests. |
| **Admin / Moderator** | A registered user with elevated privileges. Can hide, archive, or delete inappropriate requests. Managed via a role field (no separate admin panel in V1). |

## User Stories

| # | Story | Priority |
| :--- | :--- | :--- |
| **US-1** | As a guest, I can view the prayer wall and see all open requests sorted newest-first, so I can immediately see what others need prayer for. | [Must-Have] |
| **US-2** | As a guest, I can post an anonymous prayer request (text only), so I can ask for prayer without creating an account. | [Must-Have] |
| **US-3** | As a guest or member, I can tap "I prayed" on any request and see the count increment immediately, so the requester knows someone prayed. | [Must-Have] |
| **US-4** | As a visitor, I can sign up with email + password, so I can have a persistent identity and receive notifications. | [Must-Have] |
| **US-5** | As a member, I can log in and log out, so my session is persistent across visits. | [Must-Have] |
| **US-6** | As a member, I can post a prayer request and choose whether it appears as "anonymous" or with my display name. | [Must-Have] |
| **US-7** | As a member, I can receive a web push notification when someone prays for my request, so I feel supported. | [Nice-to-Have] |
| **US-8** | As a member, I can mark my own request as "answered", so the community can celebrate with me. | [Nice-to-Have] |
| **US-9** | As an admin, I can hide or delete an inappropriate request from the wall. | [Must-Have] |
| **US-10** | As a user, I can install the app to my home screen and access it like a native app. | [Nice-to-Have] |
